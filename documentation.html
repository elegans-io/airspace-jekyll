How to use ORAC, in details

Why ORAC is different

ORAC is more than a recommendation engine, and is more than a tool for personalization. ORAC learns --by sending slightly different messages to different users, and observing the reactions.

Such technique, a "generalized A/B testing", if conducted properly, eliminates the need to run several sequential A/B tests.

Downsides of this technique are its requirements in terms of statistical analysis skills --which is the reason why we like it.

  • Greetings
  • History

  • Greetings

    {greetings}
    Different greetings expressions (Dear {name}, Hello...).

    {farewell}
    Farewell expressions (Bye, Regards...)

  • History

    {surf_history}
    Array of last visited object (pages, items...). Object can be defined by the customer, e.g.

    {surf_history.items}
    Last visited products (with name, url, image, price...)

    {surf_history.pages}
    Visited pages (with title, url...)

  • Recommendations

    {recommendations}
    Array of recommended object. As for history, object are defined by the customer, e.g.

    {recommendations.items}
    Recommended items (with .name, .url, .image, .price...)

    {recommendations.users}
    E.g. for social networks, users you might know (with title, url...)
